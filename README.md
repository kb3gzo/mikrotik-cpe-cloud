# Mikrotik CPE Cloud

Central management + telemetry for the Bradford Broadband Mikrotik hAP fleet.

This repository is the implementation of the designs in `01-design-wireguard-and-telemetry.md` and `02-self-provisioning.md`. Phase 1 scope is: WireGuard overlay, push-based telemetry ingest, zero-touch self-enrollment, and an admin web UI to approve pending routers.

## Production install on Ubuntu 22.04

For a fresh droplet at `mcc.bradfordbroadband.com`, follow the full runbook in **[INSTALL.md](INSTALL.md)** — it covers apt repos for Postgres 15 + InfluxDB 2 + Python 3.11 (via deadsnakes), WireGuard server keypair + `wg0.conf`, nginx + Let's Encrypt TLS, the systemd unit, UFW, the unprivileged `cpecloud` service account, and the sudo-wrapped `wg syncconf` helper. Top-to-bottom, it's ~15 minutes of paste-and-verify.

## Local development

Prereqs: Python 3.11, Postgres 15, InfluxDB 2.x (optional for migrations-only work).

```bash
# 1. Virtualenv + deps
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. Env file
cp .env.example .env
# Edit DATABASE_URL, INFLUX_*, WG_* as appropriate

# 3. Create the database + user
sudo -u postgres psql <<'SQL'
CREATE USER cpecloud WITH PASSWORD 'cpecloud';
CREATE DATABASE cpecloud OWNER cpecloud;
SQL

# 4. Run the initial migration
alembic upgrade head

# 5. Boot the API
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
# Visit http://127.0.0.1:8000/docs for the OpenAPI explorer
# Visit http://127.0.0.1:8000/healthz for a liveness probe
```

At this point the HTTP endpoints (`/factory/self-enroll.rsc`, `/api/v1/enroll`, `/api/v1/auto-enroll`, `/api/v1/telemetry`) all return HTTP 501 — they're wired into FastAPI but the handlers are stubs. Deliverables #3–#7 in `01-design §12` fill them in.

## Migration workflow

Autogenerate a new revision after editing models in `app/models/`:

```bash
alembic revision --autogenerate -m "short summary"
# Review the generated file under alembic/versions/ before committing.
```

The initial revision (`0001_initial_schema.py`) is hand-written rather than autogenerated, because the trigram index and seed rule need explicit SQL that autogenerate doesn't produce.
